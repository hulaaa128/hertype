# HerType · 父权语言拼写检查器

> 像查拼写错误一样，查出语言里的父权惯性。

HerType 是一个**纯前端**的中文语言检查工具。它不纠正错别字，而是像拼写检查器一样，在你写下的句子底下划出那些**习以为常、却在悄悄规训性别的表达**——并告诉你它为什么值得被看见。

理论框架参考 Amanda Montell 的《Wordslut: A Feminist Guide to Taking Back the English Language》（中译《语言恶女》）。

## 这不是又一个 AI 对话框

和「直接问 AI 有没有性别问题」相比，HerType 的价值在三处别人给不了的体验：

- **环境式提醒**：你不用提问，它像 Word 抓错别字一样，主动在你写的字底下划红线。
- **镜像模式（核心）**：把一句话性别对调，让你亲眼看见「同样的话对着男性说时有多刺耳」。不指责，只呈现——共情在那个「不对劲」的瞬间自然发生。
- **机制解释**：每处命中都摊开它背后的父权语言机制，而不只是打个标签。

它的立意不是制造对立，而是让语言回到就事论事。

## 三种模式

| 模式 | 作用 |
| --- | --- |
| **学习模式** | 本地词库即时标注 + AI 按机制识别更隐蔽的规训表达，摊开每处背后的机制 |
| **输出模式** | 把带性别标注/预设的句子改写得更平等、就事论事（如「女程序员好酷」→「程序员好酷」） |
| **镜像模式** | 性别对调，让你直观感受语言里的双重标准 |

## 识别机制（源自《Wordslut》框架）

母职/亲属羞辱 · 性经验羞辱 · 女性气质贬低 · 女性竞争污名 · 外貌规训 · 能力性化否定 · 性别角色规训 · 语义贬降 · 非人化物化 · 阳性默认/称谓降格

其中「听起来正常、实则规训」的隐蔽表达（如「女生就该顾家」「男生就要有担当」）由 AI 按语义识别，本地固定词库难以覆盖。

## 使用

纯静态页面，克隆后本地起个服务器即可：

```bash
python3 -m http.server 8000
# 浏览器打开 http://localhost:8000
```

想用 AI 能力（机制识别 / 重写 / 镜像对调），点右上角 **⚙ 设置**，填入任意 OpenAI 兼容接口的 API Key、接口地址、模型。

## 隐私

- 三个模式的**本地词库标注**完全在浏览器运行，不上传。
- AI 相关功能（机制识别 / 重写 / 对调）会把当前文本发送给**你自己配置的**大模型接口。
- 你的 API Key **只存在本浏览器的 localStorage**，不写入代码、不上传任何服务器。

## 技术栈

原生 HTML / CSS / JavaScript，零框架、零构建、零后端。

- `index.html` — 页面结构
- `style.css` — 样式（克制的文学感基调）
- `lexicon.js` — 本地词库与分类引擎
- `ai.js` — OpenAI 兼容接口调用（含 SSE 流式重写）
- `app.js` — 标注引擎、镜像层叠、三模式交互
- `docs/` — 《Wordslut》机制框架整理
